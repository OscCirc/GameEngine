# 预编译头文件 (Precompiled Headers)

为了加速大型 C++ 项目（如本游戏引擎）的编译速度，我们在系统中引入了**预编译头文件 (PCH)** 机制。

## 1. 为什么需要 PCH？

在 C++ 中，每次编译一个 `.cpp` 文件时，编译器都需要将被 `#include` 的头文件内容完全展开并解析。对于一些庞大且极少变动的标准库（如 `<string>`, `<vector>`, `<functional>`）或第三方库头文件（如涉及到 Windows API 或底层库的内容），这种重复解析会严重拖慢编译速度。

通过引入 PCH：
- 编译器只需将这些不常变动的头文件编译一次，保存为特定的中间结果。
- 其他源文件包含 PCH 时，直接读取中间结果，大幅缩短编译时间。

## 2. 核心文件结构

我们引入了以下两个核心文件来管理 PCH：

*   **`src/pch.h`**
    *   **作用**：集中存放所有引擎内部需要的高频标准库包含（如 `<iostream>`, `<string>`, `<sstream>`, `<vector>`, `<functional>` 等），以及某些稳定的系统 API 接口头文件。
    *   **规范**：只有那些**变化极少**的库才适合放入此文件。如果将经常变动的头文件放入其中，任何一次修改都会导致整个项目全部重新编译，得不偿失。

*   **`src/pch.cpp`**
    *   **作用**：为了给编译器（如 MSVC）提供一个生成 `.pch` 二进制预编译文件的基础源文件。在 Visual Studio 工程中，该文件的编译选项会被单独设置为“创建预编译头 (Create)”，而其他源文件为“使用预编译头 (Use)”。

## 3. 构建系统的集成

在我们的 `premake5.lua` 脚本中，已对其进行配置，自动为不同环境生成正确的 PCH 构建设置：

```lua
project "GameEngine"
    -- ...
    pchheader "pch.h"
    pchsource "GameEngine/src/pch.cpp"
    -- ...
```
这段脚本保证了在生成 Visual Studio 项目解决方案（ `.vcxproj` ）时，编译器属性已经正确配置了 PCH。

## 4. 编码规范：如何使用 PCH？

在项目中使用 PCH，必须遵守以下约定：

1.  **首行包含**：在所有的 `.cpp` 源文件中，**第一行必须包含预编译头 `pch.h`**（如 `#include "pch.h"`）。如果不作为第一行包含，编译器（特别是 MSVC）将会报编译错误或忽略该行之前的代码。
2.  **移除冗余引用**：以前在各个类的头文件（`.h`）或源文件（`.cpp`）散落的标准库头文件引用（如 `<sstream>`, `<string>`），都可以被清理或删除，统一放置在 `pch.h` 内。
3.  **头文件无需包含**：请注意，仅在 `.cpp` 中进行第一行引入即可，引擎的其他自建头文件（`.h`）不需要显式 `#include "pch.h"`，因为使用该头文件的 `.cpp` 已经包含了 PCH。
